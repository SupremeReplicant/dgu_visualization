# DGU Diagnostic Visualization

Python приложение для визуализации и диагностики компонентов дизель-генераторной установки (ДГУ) на основе данных тепловизионного контроля.

## Описание

Приложение позволяет:
- Визуализировать 3D модель ДГУ с 33 компонентами
- Загружать и анализировать температурные данные из файлов или фотографий
- Отображать тепловую карту компонентов в реальном времени
- Проверять температуры на соответствие нормальным значениям
- Выделять проблемные компоненты в режиме осмотра
- Просматривать детальную информацию о каждом компоненте

## Требования

- Python 3.8+
- PyVista (3D визуализация)
- PyTorch и TorchVision (детекция перекрестий)
- NumPy, Pillow (обработка данных и изображений)
- Tkinter (GUI)

## Использование

### Режим 1: Осмотр модели с подсказками
- Осмотр 3D модели ДГУ
- Наведение на компонент показывает его название
- Нажатие на компонент открывает детальную информацию
- Проблемные компоненты (превышение температуры) выделены красным
- Нажатие на кнопку рядом в названием группы переключает видимость этой группы

### Режим 2: Модель с температурной картой
- Отображение 3D модели с цветовой шкалой температур
- Видимость компонентов управляется чекбоксами
- Отображение текущих температур
- Интерпретация:
  - Синий: < 30°C
  - Фиолетовый: 30-40°C
  - Зелёный: 40-50°C
  - Жёлтый: 50-60°C
  - Красный: > 60°C

### Режим 3: Таблица сравнения
- Отображение всех 29 компонентов с температурами
- Сравнение с эталонными значениями
- Общий отчёт диагностики

## Загрузка данных

### Из файла
- Текстовый файл с 29 значениями температур (одно на строке)

### Из фотографий
- Папка с 29 фотографиями (формат: PNG, JPG, JPEG)
- На каждой фотографии должно быть одно перекрестие

## Известные ограничения

- Поддерживается только Windows (hardcoded путь к шрифтам Arial)
- 3D модели должны быть в формате OBJ
- Требует GPU для оптимальной работы детекции (CPU работает медленнее)
- Многопоточность при загрузке окон может вызвать проблемы на старых машинах
